Решение задачи с помощью вычислительной техники


Два варианта определения решаемости задачи:

  1. Задача может быть решена, если можно сформировать алгоритм ее решения.
  2. Можно решить задачу, если можно доказать, что алгоритм ее решения может быть создан.

Этапы создания:

  1. Идея от заказчика (литературная постановка задачи)
  2. Анализ задачи или обследование. Цель – получить недостающую информацию.
    • Исходные данные и их характеристики.
    • Результаты.
    • Промежуточные данные – данные, которые являются результатами на одном шаге и исходными для другого шага. Информационные потоки.
    • Алгоритм решения задачи. Заказчик рассказывает про существующий алгоритм, который его не устраивает.
  3. Постановка задачи - Техническое Задание (ТЗ)
    • Расшифровка всех терминов.
    • Перечень всех ограничений.
    • Выяснить соглашения при решении задачи.
  4. Построение математической модели задачи.
  5. Замысел.
    • Перечислить способы решения задачи.
    • Проанализировать способы и найти единственный наиболее приемлемый вариант.
  6. Создание нового алгоритма. Приемы:
    • Декомпозиция.
    • Итерация.
    • Рекурсия.
    • Эвристика.
  7. Выбор языка программирования.
  8. Ручная проверка программы.

Этапы прохождения задания через вычислительную технику (ВТ):

  1. Перенос программы на ВТ.
  2. Трансляция программы с языка высокого кода на язык машинного кода.
    • Компиляторы.
    • Интерпретаторы.
  3. Редактирование связей (сборка программы).
  4. Выполнение программы.
