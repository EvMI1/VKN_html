Решение задач с помощью рекурсии

Особенности реализации

  • Реализация на ЯП выполняется только с помощью подпрограмм (функции или процедуры).
  • Когда выполняется реализация внутри подпрограммы должно быть условие, на основании которого выполняется решение, будет ли продолжаться рекурсия, либо она прекращается. Внутри подпрограммы должна быть:
    • Рекурсивная ветвь (часть программы, в которой происходит обращение программы к самой себе).
    • Не рекурсивная ветвь (когда условие не выполняется).
  • Существует два варианта рекурсии:
    • Прямая рекурсия – происходит обращение подпрограммы к этой же подпрограмме.
      Пример: A → A
    • Косвенная рекурсия – обращение к себе через другие функции.
      Пример: A → B → A
  • Прежде чем выполнять рекурсию, нужно оценить, какое количество ресурсов вычислительной техники будет затрачивать на выполнение задачи.

    def factorial(n):
        if n <= 1:  # Условие для прекращения рекурсии
            return 1
        else:
            return n * factorial(n - 1)  # Прямая рекурсия

Для реализации рекурсия вычислительная техника формирует рекурсивный стек. Это область оперативной памяти, куда записывается вся текущая информация для обеспечения работы рекурсивных вычислений. Туда пишутся параметры, все локальные переменные, резервируется место для всех результатов, адрес возврата.

В конечном итоге рекурсивный стек будет уничтожен.
Процесс выполнения рекурсивной подпрограммы до момента попадания на нерекурсивную ветвь называется прямой ход рекурсии, а после попадания на нерекурсивную ветвь – обратный ход рекурсии.
